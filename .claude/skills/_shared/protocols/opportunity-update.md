# Opportunity Update

Common patterns for updating the living `opportunity.md` one-pager.

- **Decisions table:** append a row per resolved decision — `When: /N {skill}` (or `Cascade`), the decision, and a one-line rationale. Append-only.
- **Scoring section:** keep Impact / Complexity / Confidence current as evidence arrives.
- Each skill writes its section; never overwrite another skill's section. Keep it builder-readable: an engineer should be able to start from this doc alone.
