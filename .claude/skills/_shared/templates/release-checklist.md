# Release Checklist Template

Output template for `pipeline/{idea-slug}/ship/release-checklist.md`. Pull
specific metrics from `assess.md` and dependencies from `prove.md` — don't leave
the brackets generic.

---

```markdown
## Release Checklist: [idea-slug]

### Pre-build
- [ ] Success criteria locked (KRs + targets from assess.md)
- [ ] Instrumentation plan for each KR ([metric] → [event/source])
- [ ] Kill criteria agreed with [stakeholder]
- [ ] Dependencies confirmed available (from prove.md): [dep], [dep]

### During build
- [ ] Each KR has a measurement point wired before launch
- [ ] Error, empty, and loading states designed (not just the happy path)
- [ ] Guardrail metrics instrumented ([metric] must not regress)

### Pre-launch
- [ ] Baseline captured for every KR ([metric] = [value])
- [ ] Holdout / control defined if measuring lift
- [ ] Rollback plan + owner
- [ ] Kill-criteria alert thresholds set

### Post-launch
- [ ] Day-[N] read against targets
- [ ] Kill criteria checked ([condition] within [timeframe])
- [ ] Decision logged to decision-log.md (continue / iterate / kill)

---
*Generated by /5 ship, {date}*
*Metrics from assess.md · dependencies from prove.md*
```
