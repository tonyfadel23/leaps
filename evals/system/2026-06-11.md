### Handoff Gap Detail

**explore → define target correction**: When /4 explore invalidates a /3 define target (as seen in pharmacy-auto-refill where 30K MAU was rejected), the correction is only logged in decision-log.md. Neither explore SKILL.md nor define SKILL.md has a protocol for /4 to update /3's targets. The cascade protocol (cascade.md) only warns about staleness — it doesn't prescribe which skill updates which file. This creates a live inconsistency: define.md and opportunity.md show targets the PM has already rejected.

**Recommendation:** Add a "Target Revision" protocol to explore SKILL.md Phase 4 (Synthesize): when a define.md target is invalidated during /4, update the target in both define.md and opportunity.md with the revised value + `[revised by /4, {date}]` tag.
